# 脚本文件夹管理与市场设计

**日期:** 2026-03-20
**状态:** 已批准

## 概述

优化脚本文件管理，支持一级文件夹分类 + 标签系统，预留脚本市场（中央仓库模式）的扩展能力。

## 需求摘要

| 决策项 | 结论 |
|--------|------|
| 文件夹层级 | 一级文件夹 + 标签 |
| 市场模式 | 中央仓库（发布/下载） |
| 下载行为 | 先下载副本，预留订阅更新 |
| 审核机制 | 先无审核，后期加 |
| 后端存储 | 暂不确定，接口抽象优先 |
| 前端风格 | 树形侧边栏 + 脚本列表 |
| 实现方案 | 文件系统驱动 + JSON 元数据 |

## 一、数据模型与后端架构

### 目录结构

```
scripts/
├── .scripts-meta.json          ← 元数据配置（标签、描述、市场来源）
├── 爬虫/
│   ├── scraper.py
│   └── scraper_draft.py
├── 数据处理/
│   └── csv_cleaner.py
└── _unsorted/                  ← 默认文件夹，存放未归类脚本
    └── hello.py
```

### `.scripts-meta.json` 结构

```json
{
  "version": 1,
  "scripts": {
    "爬虫/scraper": {
      "tags": ["电商", "淘宝"],
      "description": "淘宝商品数据采集",
      "source": null
    },
    "数据处理/csv_cleaner": {
      "tags": ["CSV", "清洗"],
      "description": "CSV 文件清洗工具",
      "source": {
        "market_id": "abc123",
        "author": "zhangsan",
        "downloaded_at": "2026-03-20T10:00:00",
        "version": "1.0.0"
      }
    }
  },
  "folders": {
    "爬虫": { "icon": "spider", "order": 0 },
    "数据处理": { "icon": "database", "order": 1 }
  },
  "tags": ["电商", "淘宝", "CSV", "清洗", "社交"]
}
```

**关键设计决策：**
- 脚本 key 为 `文件夹/脚本名` 格式，保持唯一性且反映层级
- `source` 字段预留市场来源信息，为后期订阅更新做准备
- `folders` 记录文件夹图标和排序，支持自定义
- `tags` 全局标签池，便于前端自动补全

### 后端 API

| 端点 | 说明 |
|------|------|
| `GET /api/scripts` | 返回树形结构（含文件夹 + 脚本 + 标签） |
| `POST /api/folders` | 创建文件夹 |
| `PUT /api/folders/{name}` | 重命名文件夹 |
| `DELETE /api/folders/{name}` | 删除文件夹（需空） |
| `POST /api/scripts/{folder}/{name}/move` | 移动脚本到其他文件夹 |
| `PUT /api/scripts/{folder}/{name}/tags` | 更新脚本标签 |
| `PUT /api/scripts/{folder}/{name}/description` | 更新脚本描述 |
| `POST /api/scripts/{folder}/{name}/run` | 运行脚本 |
| `POST /api/scripts/{folder}/{name}/stop` | 停止脚本 |
| `GET /api/scripts/{folder}/{name}/content` | 获取脚本内容 |
| `GET /api/market/scripts` | 浏览市场脚本（预留） |
| `POST /api/market/scripts/{id}/download` | 下载市场脚本（预留） |
| `POST /api/market/scripts/publish` | 发布脚本到市场（预留） |

## 三、数据流与错误处理

### WebSocket 变更事件

现有 WebSocket 只广播运行日志和状态，新增文件系统变更事件：

```json
{ "type": "folder_created", "folder": "爬虫" }
{ "type": "folder_deleted", "folder": "爬虫" }
{ "type": "script_moved", "script": "scraper", "from": "_unsorted", "to": "爬虫" }
{ "type": "tags_updated", "script": "爬虫/scraper", "tags": ["电商", "淘宝"] }
```

保证多窗口状态同步，也为后期多用户协作打基础。

### 错误处理

| 场景 | 处理方式 |
|------|---------|
| 创建已存在的文件夹 | Toast 提示「文件夹已存在」，阻止创建 |
| 删除非空文件夹 | 弹窗提示「请先移走 N 个脚本」，不允许强制删除 |
| 移动脚本时目标有同名文件 | 弹窗确认「目标文件夹已有同名脚本，是否覆盖？」 |
| 移动正在运行的脚本 | 允许移动（仅文件系统操作），运行不受影响 |
| `.scripts-meta.json` 损坏 | 后端启动时校验，损坏则从文件系统重建（标签/描述丢失，结构保留） |
| 文件夹名非法字符 | 前端输入时实时校验，只允许中文/英文/数字/下划线/连字符 |
| 外部直接操作 scripts/ 目录 | 刷新时 scanner 以文件系统为准，orphan 元数据自动清理 |

### 向后兼容

- 根目录脚本自动归入 `_unsorted/` 文件夹
